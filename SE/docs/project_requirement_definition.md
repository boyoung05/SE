# **1\. 서론** 
**1.1 문서의 목적 및 범위**

본 요구사항 정의서는 웰폰키즈(Wellness Phone Kids) 프로젝트의 요구사항을 정의하기 위한 문서이다.

본 문서에는 시스템이 제공해야 하는 기능, 성능, 보안, 사용성, 인터페이스, 개발 환경 및 기타 고려사항을 정의한다. 이를 통해 구현 범위를 명확히 하고, 프로젝트 수행 중 요구사항 변경이나 기능 누락의 가능성을 줄일 수 있다.

## **1.2 프로젝트 개요**

### **1.2.1 프로젝트 정의**

웰폰키즈는 아동의 스마트폰 사용 시간, 사용 패턴, 앱 사용 현황 등을 기반으로 보호자에게 자녀의 디지털 습관 정보를 제공하고, 아동에게는 스스로 스마트폰 사용을 조절할 수 있도록 코칭 메시지와 목표 달성 기능을 제공하는 서비스이다.

이 서비스는 단순히 스마트폰 사용을 제한하는 것이 아니라, 아동이 자신의 사용 습관을 인식하고 점진적으로 개선할 수 있도록 돕는 것을 목표로 한다.

### 

### **1.2.2 주요 기능 설명**

| ID | 기능 | 설명 |
| :---: | :---: | ----- |
| F01 | 스마트폰 사용 현황 확인 | 일간 또는 주간 단위로 스마트폰 사용 습관 리포트 제공 |
| F02 | 보호자 코칭 가이드 | 아동에게 어떤 방식으로 지도하면 좋을지 코칭 메시지를 제공 |
| F03 | 목표 설정 | 보호자와 아동이 함께 스마트폰 사용 목표를 설정 |
| F04 | 미션 및 보상 | 아동이 사용 습관 개선 미션을 수행하고 긍정적 피드백을 받음 |
| F05 | 알림 | 과도한 사용, 목표 달성, 미션 완료 등의 상황을 알림 |
| F06 | 사용자 관리 | 보호자 계정, 아동 계정, 알림 설정, 개인정보 설정 등을 관리 |

**1.3 용어 정의**

본 문서의 이해를 돕기 위해 사용된 용어 및 약어를 설명하고 정의합니다,

| 용어 | 설명 |
| :---: | :---: |

| PR | Pull Request의 약자로, 작업한 내용을 병합하기 전에 검토 요청하는 절차 |
