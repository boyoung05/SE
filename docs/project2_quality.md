# Project II - 대상 시스템 품질 요소 추정하기

## 대상 시스템
아이의 건강한 디지털 습관 코칭 서비스

## 1. 사용자가 원하는 기대치
이 시스템의 주요 사용자는 아동과 부모이다. 따라서 두 사용자 집단이 기대하는 품질을 함께 고려해야 한다.

### 1-1. 아동 사용자가 원하는 기대치
- 화면이 어렵지 않고 직관적이어야 한다.
- 글이 너무 많지 않고 이해하기 쉬워야 한다.
- 미션 수행 과정이 재미있고 성취감을 느낄 수 있어야 한다.
- 보상이나 캐릭터 성장 요소가 흥미롭게 제공되어야 한다.
- 앱 사용 과정에서 복잡한 절차 없이 쉽게 참여할 수 있어야 한다. 

### 1-2. 부모 사용자가 원하는 기대치
- 자녀의 스마트폰 사용 습관을 한눈에 쉽게 파악할 수 있어야 한다.
- 주간 리포트, 감정 체크, 미션 수행 결과 등이 명확하게 정리되어야 한다.
- 아이를 지나치게 통제하지 않으면서도 건강한 습관 형성을 도울 수 있어야 한다.
- 부모 코칭 가이드가 실제 생활에서 활용 가능해야 한다.
- 자녀의 사용 데이터와 감정 정보가 안전하게 보호되어야 한다.

## 2. 개발자 혹은 유지보수 담당자가 원하는 기대치
개발자와 유지보수 담당자는 시스템이 지속적으로 운영되고 개선될 수 있도록 다음과 같은 품질을 기대한다.

- 기능별 구조가 명확하게 분리되어 있어야 한다.
- 기능이 모듈화되어 있어야 한다.
- 새로운 미션이나 보상 체계를 쉽게 추가할 수 있어야 한다.
- 특정 기능 수정이 다른 기능에 큰 영향을 주지 않아야 한다.
- 오류 발생 시 원인을 빠르게 찾을 수 있어야 한다.
- 테스트 코드 작성이 쉬운 구조여야 한다.
- 서비스 관련 정보가 문서화가 잘 되어 있어야 한다.

## 3. 개발 조직의 관리자가 원하는 기대치
개발 조직의 관리자는 일정, 비용, 품질, 서비스 성장 가능성 관점에서 시스템을 바라본다.

- 개발 일정이 현실적으로 계획되고 예측 가능해야 한다.
- 초기 버전에서 핵심 기능을 우선 구현할 수 있어야 한다.
- 개발 비용 대비 효과가 커야 한다.
- 서비스 품질 문제가 적어 운영 리스크가 낮아야 한다.
- 사용자의 만족도와 지속 사용 가능성이 높아야 한다.
- 차후 기능 확장이나 사업화 가능성이 있어야 한다.
- 사회적 문제 해결 서비스로서 의미와 차별성이 분명해야 한다.
- 개인정보 보호와 같은 법적/윤리적 문제를 안정적으로 관리할 수 있어야 한다.
